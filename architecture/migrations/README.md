# 🚀 مجلد ترحيل وترقية قاعدة البيانات (Database Migrations)

## 🎯 هدف المجلد
هو المكان الرسمي المخصص لحفظ سكربتات الترحيل والتعديل الهيكلي الفعلي (DDL & DML) لقاعدة البيانات. كل ملف ترحيل يمثل خطوة معمارية حذرة ومحددة بدقة تابعة لإحدى مراحل خارطة طريق الدستور الأعلى V5، مما يضمن الترقية الآمنة والمنظمة والمسجلة للتعديلات.

## 🛠️ وظيفة الملفات الموجودة بداخله
* **ملفات الترحيل المعنونة بالمراحل** (مثال: `V5-PHASE-01.sql`, `V5-PHASE-02.sql`): تحتوي على التعليمات الفعلية الحية لتطوير الهياكل والسياسات الخاصة بكل مرحلة على حدة في قاعدة البيانات.

## ✅ ما يجوز وضعه بداخله
* سكربتات التعديل والإنشاء والحذف المحدودة والمنضبطة مثل:
  * `CREATE TABLE`, `ALTER TABLE`
  * `CREATE POLICY`, `DROP POLICY`
  * `CREATE INDEX`, `DROP INDEX`
  * `CREATE FUNCTION`, `CREATE TRIGGER`
* أي تعليمات DDL لتجهيز الحقول والقيود الفنية للمرحلة المحددة فقط.

## ❌ ما يُمنع وضعه بداخله
* يُمنع منعًا باتًا وضع تعليمات ترحيل عشوائية أو غير منسوبة لمرحلة معينة من خارطة الطريق.
* يُمنع وضع أي تعليمات ترحيل لخطوات مستقبلية لم يتم الاتفاق على تصميمها وتوقيع الـ ADR الخاص بها.
* يُمنع وضع أكواد اختبارات التحقق أو الفحص القائمة على القراءة فقط (مكانها مجلد `verification`).

## 🔄 ترتيب استخدامه أثناء دورة التطوير
يتم تفعيل واستخدام مجلد الـ Migrations في **منتصف الدورة التطويرية المعتمدة (`Step 4`)**:
1. بعد فحص الامتثال المسبق وثبات خط الأساس وصياغة الـ ADR وتوقيعه.
2. يتم تشغيل ملف الـ Migration المعين للمرحلة لتعديل قاعدة البيانات الحية.
3. فور اكتمال التعديل بنجاح، يتم الانتقال فوراً إلى مرحلة التحقق الفردي لنتائج هذا الملف المعين (`verification/Verify-V5-PHASE-XX.sql`).
