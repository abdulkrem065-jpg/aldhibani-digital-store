# 🔄 مجلد خطط وسكربتات التراجع (Rollback Engine)

## 🎯 هدف المجلد
هو صمام الأمان الفني وشبكة الطوارئ المخصصة لإنقاذ قاعدة البيانات وحمايتها من التلف أو فقدان الاتساق في حال فشل تطبيق أي ترحيل هيكلي. يهدف المجلد لحفظ سكربتات تراجع مخصصة وقابلة للتشغيل الفوري لكل مرحلة ترقية لضمان إمكانية العودة السليمة للحالة المستقرة السابقة بدون أي خسائر في البيانات.

## 🛠️ وظيفة الملفات الموجودة بداخله
* **ملفات التراجع المخصصة لكل مرحلة** (مثال: `Rollback-V5-PHASE-01.sql`, `Rollback-V5-PHASE-02.sql`): يحتوي كل ملف على الأوامر العكسية الدقيقة لملف الترحيل المقابل له، ليعيد النظام تماماً للحالة السابقة قبل تشغيل ملف الـ Migration.

## ✅ ما يجوز وضعه بداخله
* السكربتات التراجعية العكسية الحرة مثل:
  * `DROP TABLE`, `ALTER TABLE ... DROP COLUMN`
  * `DROP POLICY`, `DROP INDEX`
  * `DROP TRIGGER`, `DROP FUNCTION`
  * استعادة البنيات والسياسات القديمة المستقرة.

## ❌ ما يُمنع وضعه بداخله
* يُمنع منعًا باتًا وضع تراجعات هيكلية مدمرة تقوم بحذف البيانات الحية للأعمدة دون أخذ نسخ احتياطية مؤقتة أولاً.
* يُمنع وضع أي ملفات ترحيل أو ترحيل ترقية.

## 🔄 ترتيب استخدامه أثناء دورة التطوير
يمثل هذا المجلد **الخطة الاحتياطية الفورية للطوارئ (`Emergency Loop`)**:
1. في حال تشغيل الترحيل (`V5-PHASE-XX.sql`) وظهور خطأ أو فشل في التحقق الفردي (`Verify-V5-PHASE-XX.sql`) أو رصد مخالفة في فحص الامتثال الكلي.
2. يتم تشغيل ملف التراجع المقابل فورا (`Rollback-V5-PHASE-XX.sql`).
3. بعد تشغيل التراجع، تتم إعادة تشغيل فحص الامتثال الشامل للتأكد من عودة قاعدة البيانات لخط الأساس المستقر والسليم تماماً قبل البدء بتصحيح الأخطاء وإعادة المحاولة.
