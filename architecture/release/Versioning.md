# 🏷️ وثيقة معايير الترميز وإصدار النسخ (Semantic Versioning - Architecture V5)

تعتمد المنصة الفيدرالية الموحدة المعيار العالمي الصارم لترميز الإصدارات المعروف بالترميز الدلالي (**Semantic Versioning - SemVer 2.0.0**)، والذي ينظم كيفية تتبع التعديلات البنائية والبرمجية لقاعدة البيانات والتطبيقات الملحقة بها.

يتم تمثيل كل رقم إصدار بالصيغة القياسية التالية:

```text
MAJOR.MINOR.PATCH
(الرقم الرئيسي . الرقم الفرعي . الرقم التصحيحي)
```

---

## 🏗️ 1. الرقم الرئيسي (MAJOR Version)

يتم زيادة الرقم الرئيسي (`+1.0.0`) عند حدوث تغييرات معمارية جوهرية أو ترقيات هيكلية غير متوافقة مع الإصدارات السابقة (Breaking Changes). تشمل هذه الحالات:

* **تعديل أو حذف حقول وجداول أساسية:** حذف أي جدول تشغيلي أو دمج أعمدة تخل بالاستعلامات الحالية للشركات المستأجرة.
* **تعديل دستوري أو تغيير السيادة (Constitution Change):** تعديل أي من القوانين والسياسات العليا المنصوص عليها في الدستور الخامس للبروتوكول الفيدرالي.
* **تعديل قواعد عزل البيانات والمستأجرين (Isolation Changes):** تغيير هيكلية تصفية وحجب الصفوف (RLS) أو تغيير المعرفات الفريدة للمنظمات والموظفين.
* **تغيير محرك قاعدة البيانات أو الترقيات الكبرى:** الانتقال لإصدار رئيسي جديد لقاعدة البيانات (مثل ترقية خادم PostgreSQL من 15 إلى 17) أو تغيير أدوات ومترجمات الـ ORM المعتمدة.

---

## ➕ 2. الرقم الفرعي (MINOR Version)

يتم زيادة الرقم الفرعي (`0.+1.0`) عند إضافة وظائف أو كائنات جديدة متوافقة بالكامل مع الإصدارات السابقة ولا تؤدي لتعطل الاستعلامات والعمليات الحالية (Backward-Compatible Additions). تشمل هذه الحالات:

* **إضافة جداول أو مجالات جديدة:** ترحيل مجالات وكتالوجات أعمال اختيارية تابعة لمانيفست معتمد (مثال: إضافة جداول نقاط البيع أو ذواكر الذكاء الاصطناعي دون التعديل على الجداول التأسيسية القائمة).
* **إضافة حقول غير إلزامية:** إلحاق أعمدة جديدة تقبل القيم الفارغة (`NULLABLE`) أو تحتوي على قيم افتراضية معززة لتجنب توقف الإدخال الحالي.
* **إضافة دوال ومستندات معرفية مرافقة:** إلحاق دوال بعيدة (RPC) إضافية، أو مشغلات إضافية مرافقة لتتبع حركات إضافية، أو إدخال قوالب وسياقات جديدة للوكيل الذكي.
* **إضافة فهارس تحسين الأداء (Performance Indexes):** تشييد فهارس إضافية لتسريع استعلامات التقارير المالية والضريبية.

---

## 🔧 3. الرقم التصحيحي (PATCH Version)

يتم زيادة الرقم التصحيحي (`0.0.+1`) عند إجراء تعديلات طفيفة أو سد فجوات برمجية وتصحيح أخطاء تشغيلية لا تمس الهيكل الأساسي ولا تضيف كائنات جديدة (Backward-Compatible Bug Fixes). تشمل هذه الحالات:

* **تصحيح فجوات أو أخطاء برمجية (Bug Fixes):** تعديل دالة التحقق الحسابي لضمان توازن القيود أو إصلاح صياغة مشغل التاريخ التلقائي للفرع.
* **ضبط وترقيع السياسات الأمنية (Policy Adjustments):** تصحيح صياغة شرط التحقق في سياسة الـ RLS لجداول المبيعات لمنع بطء المعالجة أو لتصحيح صلاحية دور وظيفي بالـ RBAC.
* **تعديل أوصاف وسجلات التوثيق والميتاداتا:** تحديث ملفات التوضيح والأوصاف الرقمية للجداول والأعمدة في كتالوج قاعدة البيانات.
* **تثبيت ترقيعات برمجية للمكتبات والامتدادات (Security Patches):** ترقية الامتدادات البرمجية (مثل تحديث إصدار `pgvector` من 0.5 إلى 0.5.1) لمعالجة مشكلات الاستقرار وسرعة حساب المسافات اللغوية.
