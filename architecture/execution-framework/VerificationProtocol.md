# 🧪 بروتوكول التحقق والفحص الهيكلي (Verification Protocol - V5)

يوثق هذا البروتوكول الإجراءات والخطوات الفنية الإلزامية التي يخضع لها نظام قاعدة البيانات فور الانتهاء من تشغيل أو تطبيق أي ملف ترحيل (Migration File) للتحقق من سلامة البنية الناتجة وامتثالها التام لشروط وضوابط المعمارية الخامسة.

---

## 🚦 الأركان الأربعة للفحص والتدقيق الفيدرالي (The Four Pillars of Verification)

لا تعتبر أي عملية ترحيل أو مهاجرة ناجحة ومكتملة ويمنع ترحيل المانيفست التالي لها إلا إذا اجتازت الفحوصات الأربعة التالية وحصلت على علامة نجاح كاملة (Passed):

```text
┌────────────────────────────────────────────────────────────────────────┐
│                        الأركان الأربعة للتحقق                          │
├───────────────────┬───────────────────┬────────────────┬───────────────┤
│    1. الهيكلي     │    2. الوظيفي     │   3. الأمني    │    4. الأداء  │
│    (Structural)   │   (Functional)    │   (Security)   │ (Performance) │
└───────────────────┴───────────────────┴────────────────┴───────────────┘
```

---

## 🔍 التفصيل الفني لخطوات الفحص والتحقق

### 1. التحقق الهيكلي والتكويني (Structural Verification)
* **الهدف:** التأكد من صياغة وتشييد الجداول والأعمدة والروابط بدقة مادية ومطابقتها التامة للمخطط العام (Blueprint).
* **خطوات الفحص:**
  * [ ] استعلام كتالوج قاعدة البيانات للتحقق من وجود الجداول والأعمدة المضافة حديثاً.
  * [ ] فحص مطابقة أنواع البيانات (Data Types) وحظر القيم الفارغة (`NOT NULL`) للحقول الإلزامية.
  * [ ] التأكد من قيام المفاتيح الأساسية (`PRIMARY KEY`) واستقرار قيود المفاتيح الأجنبية والربط البيني للمجالات.
  * [ ] موازنة وهيكلة الجداول مع المخطط لضمان عدم وجود فجوات تزيح خط الأساس الفني للخادم.

### 2. التحقق الوظيفي والحسابي (Functional Verification)
* **الهدف:** فحص استجابة الدوال المخزنة، المشغلات التلقائية، وآلات الحالات ومطابقتها للمتطلبات المالية والمخزنية المقررة.
* **خطوات الفحص:**
  * [ ] إجراء اختبارات الإدراج التلقائي للتحقق من قيام المعرفات الفريدة (UUID) والتواريخ التلقائية.
  * [ ] اختبار مشغل الحساب المالي للفواتير والتأكد من مطابقة الضرائب والإجماليات ومنع القيم السالبة.
  * [ ] فحص استجابة دالة النقل البيني والتسوية المخزنية والتأكد من خصم وإضافة الكميات بالتوازي.
  * [ ] اختبار مشغلات توازن قيود الدفتر المالي العام المزدوج وإلغاء القيود الملتوية حسابياً.
  * [ ] محاكاة الانتقال البيني لآلات الحالات لضمان حظر القفز والالتفاف المعاملي.

### 3. التحقق الأمني وعزل المستأجرين (Security Verification)
* **الهدف:** التحقق من فعالية وحصانة غلاف الحماية وقفل البيانات وحظر تسريب أو تداخل المعلومات للشركات المستأجرة.
* **خطوات الفحص:**
  * [ ] استعلام حالة الـ RLS على الجداول المضافة للتأكد من تفعيل `ENABLE ROW LEVEL SECURITY` و `FORCE ROW LEVEL SECURITY`.
  * [ ] محاكاة استعلام الموظف الحامل للـ JWT وفحص نجاح تصفية الصفوف طبقاً لهوية مستأجره الفعلي.
  * [ ] محاكاة محاولة استعلام خارجي أو عشوائي للوصول لبيانات شركة غريبة والتحقق من الرفض والحجب التلقائي للخادم.
  * [ ] اختبار قيود جدول التدقيق التاريخي `audit_log` ومحاولة تعديل أو حذف صف فيه والتحقق من إطلاق الخادم لخطأ الرفض الصارم.
  * [ ] فحص شجرة الصلاحيات والأدوار (RBAC) ومحاكاة تداول حركات حساسة خارج نطاق الرتبة الوظيفية المعتمدة للموظف.

### 4. التحقق من كفاءة وسرعة الأداء (Performance Verification)
* **الهدف:** التأكد من ثبات وسرعة استجابة قاعدة البيانات ومراقبة استهلاك الموارد وتحسين مسارات الاستعلامات.
* **خطوات الفحص:**
  * [ ] التحقق من بناء وقيام الفهارس المقترحة (`INDEXES`) والبحث السيمانتك المتجهي (`HNSW Indexes`).
  * [ ] استخدام أوامر قياس الأداء والمسارات (مثل `EXPLAIN ANALYZE`) للتحقق من عدم وجود مسح شامل وثقيل للجداول (`Seq Scan`) أثناء عمليات البحث والاستعلام الحيوية.
  * [ ] قياس زمن الاستجابة والوصول لبيانات المخزن الحي وتحديث الأرصدة وإصدار الفواتير للتأكد من بقائها في نطاق الملي ثانية المسموح تشغيلياً.
  * [ ] فحص استقرار الذاكرة وسرعة مقارنة المتجهات اللغوية وحساب مسافات الجيب تمام بقاعدة البيانات.
