# 💻 قواعد توليد وصياغة نصوص الـ SQL (SQL Generation Rules - Architecture V5)

توثق هذه الوثيقة الضوابط المنهجية والمعايير الفنية الواجب الالتزام بها من قبل مهندسي الباكيند وأنظمة الذكاء الاصطناعي (مثل Google AI Studio) عند توليد وصياغة نصوص وملفات الـ SQL لتعديل بنية قاعدة البيانات الفيدرالية الموحدة.

---

## 🔒 1. محددات وضوابط ما قبل التوليد (Pre-Generation Gate)

* **منع التوليد المبكر أو العشوائي (No Premature SQL Generation):**
  يُمنع منعاً باتاً صياغة أو كتابة أي كود SQL أو إنشاء جداول في قاعدة البيانات قبل اكتمال وصياغة واعتماد كافة الوثائق المرجعية الأساسية (الدستور، ADR، المخطط، عقود التنفيذ، وبيان المهاجرة المقابل للتعديل) بنسبة 100%.
* **المرجعية الوحيدة والأحرف القياسية (Standard Reference & Casing):**
  تتم صياغة الكلمات الدلالية لـ SQL بالأحرف الكبيرة القياسية (مثل `SELECT`, `CREATE TABLE`, `ALTER TABLE`, `ROW LEVEL SECURITY`, `CONSTRAINT`) وكتابة أسماء الجداول والأعمدة والأدوار بالأحرف الصغيرة الفصيحة وبصيغة الثعبان (snake_case).

---

## 📦 2. قواعد التقسيم وعزل المجالات (Granularity & Domain Isolation)

* **ملف ترحيل واحد لكل تغيير هيكلي (One Migration Per Single Scope):**
  يُصاغ كل تعديل هيكلي أو ترقية في ملف ترحيل مادي منفصل ومغلق النطاق. يمنع دمج تعديلات هيكلية لعدة مجالات أعمال مختلفة داخل ملف ترحيل واحد.
* **الفصل الحاسم بين البيانات والأمن والتحصين (Separation of Schema, Logic, and Security):**
  لضمان دقة التنفيذ وتفادي الفشل التشغيلي للترقيات، يجب فصل صياغة المهاجرة إلى خطوات معزولة تمنع تداخل الكود:
  1. صياغة هياكل الجداول والقيود الأساسية أولاً (`DDL Schema`).
  2. صياغة الدوال والمشغلات وآلات الحالات المرافقة ثانياً (`Procedural Logic`).
  3. تفعيل سياسات الـ RLS وحراسة الأدوار الـ RBAC والتدقيق الموحد ثالثاً (`Hardening & Security`).

---

## ⚙️ 3. موثوقية وأمان نصوص الـ SQL (Robustness & Integrity)

* **إلزامية العبور الآمن والتحققات المسبقة (Idempotency Requirement):**
  يجب أن تصاغ نصوص الـ SQL بطريقة تسمح بتشغيلها وإعادتها عدة مرات دون التسبب في حدوث خطأ أو تكرار بالخادم. تلتزم الصياغة بالتحققات التلقائية مثل `IF NOT EXISTS` عند إنشاء الجداول والفهارس، و `IF EXISTS` عند تعديل أو إسقاط الكائنات غير الأساسية.
* **الربط الإجباري للمفاتيح الأجنبية والقيود (Foreign Keys & Checks):**
  تُصاغ قيود الربط مدمجة في صياغة الجدول مع تخصيص أسماء معبرة وفريدة لكل قيد لسهولة تتبعه ومعالجته عند حدوث تعارض (مثل `fk_branch_inventory_product` أو `chk_product_positive_price`).
* **تفعيل قفل وأمان الصفوف آلياً (Automatic RLS Activation):**
  يتضمن كل ملف ترحيل ينشئ جدولاً جديداً أو يعدله بنداً صريحاً ومباشراً لتفعيل أمان الصفوف ومنعه من التجاوز الإداري:
  ```sql
  ALTER TABLE table_name ENABLE ROW LEVEL SECURITY;
  ALTER TABLE table_name FORCE ROW LEVEL SECURITY;
  ```
* **توثيق المتغيرات والميتاداتا (Self-Documenting SQL):**
  يجب إلحاق تعليقات واضحة وموثقة في أعلى كل ملف SQL تصف الكاتب، وتاريخ الصياغة، ومعرف البيان المرجعي المنسوب، ورقم الإصدار الدلالي للتحديث.
