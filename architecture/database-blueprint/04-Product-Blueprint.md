# 📦 مخطط مجال المنتجات والخدمات (Product Database Blueprint)

## 1. Purpose (الهدف الفني)
تنظيم وهيكلة الكتالوج التجاري والدليل التعريفي للمنتجات والخدمات المعروضة للبيع داخل فروع ومخازن المنظمة، وضبط تكرار الباركودات ونماذج تسعير الشراء والبيع القياسية لضمان دقة العمليات وحساب الأرباح والضرائب.

---

## 2. Tables (الجداول التابعة للمجال)
* `categories` (التصنيفات والجموع السلعية)
* `products` (المنتجات والخدمات المعرفة بالكتالوج)
* `product_barcodes` (الرموز والباركودات البديلة والوطنية للمنتجات)

---

## 3. Columns (الأعمدة وهياكلها)

### جدول `categories`
| Column | Type | Nullable | Default | Notes |
| :--- | :--- | :--- | :--- | :--- |
| `id` | UUID | No | gen_random_uuid() | Primary Key |
| `organization_id` | UUID | No | None | Foreign Key للمنظمة المستأجرة |
| `category_name` | VARCHAR(150) | No | None | اسم المجموعة السلعية بالكامل |
| `is_active` | BOOLEAN | No | TRUE | مؤشر تفعيل التصنيف التجاري |
| `created_at` | TIMESTAMPTZ | No | CURRENT_TIMESTAMP | تاريخ الإنشاء والتثبيت |

### جدول `products`
| Column | Type | Nullable | Default | Notes |
| :--- | :--- | :--- | :--- | :--- |
| `id` | UUID | No | gen_random_uuid() | Primary Key |
| `organization_id` | UUID | No | None | Foreign Key للمنظمة المستأجرة |
| `category_id` | UUID | No | None | Foreign Key للتصنيف التابع له المنتج |
| `sku` | VARCHAR(100) | No | None | كود المنتج التعريفي الخاص (SKU) |
| `product_name` | VARCHAR(200) | No | None | اسم السلعة أو الخدمة بالكامل |
| `standard_buy_price` | NUMERIC(15, 2) | No | 0.00 | السعر القياسي المقدر للشراء والتكلفة |
| `standard_sell_price` | NUMERIC(15, 2) | No | 0.00 | السعر القياسي المقدر للبيع للعملاء |
| `tax_rate` | NUMERIC(5, 2) | No | 15.00 | نسبة ضريبة القيمة المضافة الافتراضية (مثال: 15.00) |
| `is_service` | BOOLEAN | No | FALSE | مؤشر ما إذا كان المنتج خدمة بلا مخزون مادي |
| `is_active` | BOOLEAN | No | TRUE | حالة تنشيط المنتج في المبيعات |
| `created_at` | TIMESTAMPTZ | No | CURRENT_TIMESTAMP | تاريخ التأسيس الفني |

### جدول `product_barcodes`
| Column | Type | Nullable | Default | Notes |
| :--- | :--- | :--- | :--- | :--- |
| `id` | UUID | No | gen_random_uuid() | Primary Key |
| `product_id` | UUID | No | None | Foreign Key للمنتج الأصلي |
| `barcode` | VARCHAR(100) | No | None | الرمز الشريطي والباركود الفريد (EAN/UPC/Internal) |

---

## 4. Keys (المفاتيح والمؤشرات)
* **Primary Keys:**
  * `categories.id`
  * `products.id`
  * `product_barcodes.id`
* **Foreign Keys:**
  * `categories.organization_id` يشير إلى `organizations.id` (مع حذف مرجعي متتالي CASCADE).
  * `products.organization_id` يشير إلى `organizations.id` (مع حذف مرجعي متتالي CASCADE).
  * `products.category_id` يشير إلى `categories.id` (مع حذف مرجعي مقيد RESTRICT).
  * `product_barcodes.product_id` يشير إلى `products.id` (مع حذف مرجعي متتالي CASCADE).
* **Candidate/Unique Keys:**
  * `products` (مزيج `organization_id` + `sku` فريد لمنع تكرار رموز السلع داخل المستأجر الواحد).
  * `product_barcodes` (مزيج `product_id` + `barcode` فريد للمنتج، والباركود نفسه فريد على مستوى المنظمة لعدم التداخل).

---

## 5. Relationships (العلاقات ورسمها النصي)
* **علاقة (واحد إلى متعدد) من `categories` إلى `products`:** التصنيف الواحد يضم ويرتب منتجات متعددة.
* **علاقة (واحد إلى متعدد) من `products` إلى `product_barcodes`:** المنتج الواحد قد يحمل باركوداً واحداً أو باركودات بديلة متعددة لتسهيل القراءة.

```text
[ categories ] 
       1
       │
       ▼ 1..*
  [ products ] ─── 1..* ──► [ product_barcodes ]
```

---

## 6. Constraints (القيود الهندسية)
* **NOT NULL:** مفروض على كافة معرفات الهوية والأسماء وتفاصيل التسعير ونسب الضرائب والباركودات.
* **CHECK:**
  * التحقق من قيم تسعير صحيحة موجبة (`standard_buy_price` و `standard_sell_price` أكبر من أو تساوي 0.00).
  * التحقق من نسبة الضريبة المقبولة (`tax_rate` بين 0.00 و 100.00).
  * التحقق من الـ SKU للتأكد من خلوه من الفراغات والرموز الغريبة.

---

## 7. Index Strategy (استراتيجية الفهارس)
* **BTree Index:**
  * فهرس مدمج على `products(organization_id, category_id, is_active)` لتسريع استدعاء الكتالوج المعزول والمفعل.
  * فهرس فريد على `product_barcodes.barcode` لتسريع عمليات البيع المعتمدة على قارئ الباركود.
  * فهرس فريد على `products.sku` للمنظمة الواحدة.

---

## 8. Generated Columns (الأعمدة المولدة تلقائياً)
* لا توجد أعمدة مولدة حالياً في جداول المنتجات والكتالوج.

---

## 9. Computed Fields (الحقول المحسوبة برمجياً)
* `profit_margin`: حقل رياضي محتسب يستعلم ويطرح تكلفة الشراء من سعر البيع لمعرفة هامش الربح المقدر للمنتج ونسبته المئوية في التقارير التسويقية والمالية.

---

## 10. Triggers Required (المؤشرات والمشغلات المطلوبة)
* `sync_product_updated_at`:
  * **الهدف:** تحديث الحقل الزمني `updated_at` في جداول المنتجات فور قيام المسؤول بتعديل أي مواصفة أو سعر للسلعة.
  * **متى يعمل:** قبل تحديث السطر (`BEFORE UPDATE`).

---

## 11. Functions Required (الدوال المطلوبة)
* `get_product_by_barcode_or_sku`:
  * **الهدف:** توفير وسيلة بحث سريعة للكاشير والباكيند لاسترجاع المنتج فورياً بمجرد إدخال الباركود الشريطي أو رمز الـ SKU المعتمد.
  * **Input:** `org_id` (UUID), `search_term` (VARCHAR).
  * **Output:** معرف المنتج واسمه وأسعاره القياسية.

---

## 12. RLS Strategy (استراتيجية عزل الضمان وبوابة الأمان)
* **العزل حسب المستأجر (Organization Dimension):**
  * تخضع جميع جداول هذا المجال لعزل المنظمة والأمان الفيدرالي.
  * يُسمح لجميع موظفي المستأجر بقراءة الكتالوج، ويُقيد تعديله وإدارته على الإداريين وحاملي الصلاحيات المعتمدة في سجل الـ RBAC الخاص بالمنظمة.

---

## 13. Audit Requirements (متطلبات الرقابة والتدقيق التاريخي)
* يُخضع جدول `products` لعمليات الرقابة وتوثيق سجلات الرقابة فور قيام أي مستخدم بتعديل الأسعار القياسية للبيع أو الشراء، لتجنب حدوث تلاعب خفي بالأسعار في الكتالوج.

---

## 14. AI Integration (التكامل مع الذكاء الاصطناعي وحماية البيانات)
* **ما يرسل:** يرسل للذكاء الاصطناعي الاسم والكتالوج والتصنيف وأسعار المنتجات لمساعدته في تدوين تحليلات الأرباح، واقتراح خطط المبيعات وتصنيف السلع تلقائياً.
* **ما يمنع إرساله:** يمنع إرسال أي أسرار توريد خاصة بتكلفة الشراء الفعلية للشركات المنافسة في البيئات الفيدرالية المشتركة.

---

## 15. Events (الأحداث المصدرة والمستقبلة)
* **الأحداث المصدرة:**
  * `product.added` (تنبيه أنظمة الجرد لتهيئة سجل كميات صفري في كافة الفروع التابعة للمستأجر).
  * `product.price.changed` (تعديل أسعار الكتالوج لإخطار واجهات الكاشير لسرعة مزامنة الأسعار).

---

## 16. Performance Considerations (اعتبارات الأداء والتحسين الفيدرالي)
* جداول الكتالوج تعتبر عالية القراءة جداً عند كل عملية مسح باركود لدى الكاشير. ينصح بشدة بالاحتفاظ بفهرس الباركود `barcode_idx` خفيفاً ومحملاً بالكامل في الذاكرة لتلبية الاستجابة في أجزاء من الملي ثانية.

---

## 17. Future Expansion (خطط التوسيع المستقبلية والترقية الآمنة)
* دعم تعدد سمات وخيارات المنتجات (Variants: المقاس، اللون، الحجم) عبر التخطيط مستقبلاً لجدول مخصص `product_variants` يرتبط بالمنتج الأصلي ويستعرض الباركودات المخصصة لكل سمة على حدة دون كسر الهيكل الحالي.
