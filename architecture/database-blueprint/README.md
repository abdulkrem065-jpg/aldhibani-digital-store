# 📐 مخططات وهندسة قاعدة البيانات الفنية (Database Blueprints)

## 🎯 ما هو الـ Database Blueprint
الـ **Database Blueprint** هو المخطط الهندسي الفني الرسمي والوحيد لقاعدة البيانات في هذا النظام. يمثل هذا المجلد الترجمة التقنية الدقيقة من المواصفات المنطقية للمجالات (`Domain Specifications`) إلى هياكل بيانية جاهزة للتطبيق فيزيائياً في قاعدة بيانات PostgreSQL/Supabase.

لا تعتبر هذه الملفات برمجية أو ترحيلية، ولكنها تمثل المواصفات الحدية والنوعية لجميع الكيانات، والأعمدة، والمفاتيح، والعلاقات، والقيود المحاسبية والفهارس، وآليات الأمان دون التورط في تفاصيل الترحيل أو كتابة كود SQL مباشر.

---

## 🔗 علاقة الطبقة بالمكونات المعمارية الأخرى

```text
📜 Constitution (الدستور الأعلى للنظام)
         ↓
🧭 ADR (سجل القرارات المعمارية المعتمدة)
         ↓
📁 Domain Specifications (المواصفات المنطقية للمجالات)
         ↓
📐 Database Blueprints (المخططات الهندسية الفنية - هذا المجلد)
         ↓
🚀 SQL Migrations (ملفات التعديل والترقية لقاعدة البيانات)
         ↓
☑️ Verification Scripts (سكربتات التحقق الفردية)
```

1. **الدستور الأعلى و specifications:** يستلم البلوبرنت القوانين والحدود ويفصل هياكلها التقنية بما يطابق عزل المستأجرين وأمن الـ RLS والرقابة.
2. **المهاجرات والـ SQL:** تلتزم المهاجرات الفنزيلية (`Migrations`) والـ SQL بتنفيذ ما تم إقراره في هذه المخططات حرفياً دون أي اجتهاد أو زيادة أو نقصان.
3. **التحقق (Verification):** يقوم التحقق بمطابقة قاعدة البيانات مع المعايير الفنية المكتوبة في هذا الملف.

---

## 🚫 محرمات وضوابط صارمة داخل المجلد
ضماناً لنقاء وجودة التوثيق وحمايته من التداخل مع بيئات التنفيذ المباشرة:
* **يُمنع منعاً باتاً** كتابة أي تعليمة SQL أو DDL أو PL/pgSQL حية داخل هذه الملفات (ممنوع `CREATE TABLE`, `ALTER`, `CREATE POLICY`... إلخ).
* **يُمنع تماماً** إدخال أي كود برمجي تابع للباكيند أو قواعد معالجة الأخطاء السحابية.
* يتم التعبير عن الجداول والحقول وأنواع البيانات عبر الجداول التنسيقية والمواصفات المنطقية المقروءة بشرياً والقابلة للتحويل آلياً إلى SQL.
