# 📜 مجلد الدستور الأعلى للنظام (Constitution)

## 🎯 هدف المجلد
هو المستودع الرسمي والمقدس للدستور الفيدرالي المعماري الأعلى للنظام (**Architecture V5 Master Constitution**). يهدف هذا المجلد إلى حفظ المبادئ الكبرى والقواعد الحاكمة غير القابلة للكسر لجميع جوانب التصميم المعماري، والأمن، والعزل، ونطاق البيانات في قاعدة البيانات والنظام ككل.

## 🛠️ وظيفة الملفات الموجودة بداخله
* **`CONSTITUTION.md`**: يمثل وثيقة الدستور الأعلى للنظام V5 التي تصف القوانين المنطقية، ومصفوفة الهوية الخماسية، وطرق عزل المستأجرين، وحماية الـ RLS الشاملة، وسياسات بوابة العملاء، ومحرك الذكاء الاصطناعي والحوكمة، والتدقيق والرقابة الفيدرالية، وهندسة الأداء الفولاذي.

## ✅ ما يجوز وضعه بداخله
* وثائق الدستور الكلية والملحقات أو التعديلات المعمارية المصادق عليها دستوريًا.
* المبادئ الحاكمة الكبرى والقواعد الحاكمة الكلية التي تلتزم بها كافة المراحل المعمارية اللاحقة.
* القواعد الصارمة للامتثال والتحقق المعماري (مثل قاعدة التحقق من التنفيذ `Execution Validation Rule`).

## ❌ ما يُمنع وضعه بداخله
* يُمنع منعًا باتًا وضع أي ملفات برمجية أو سكربتات SQL حية للتنفيذ أو التعديل المباشر على قاعدة البيانات.
* يُمنع وضع خطط تطويرية مرنة أو قرارات فنية مؤقتة (مكانها الطبيعي هو مجلد الـ `ADR`).
* يُمنع وضع ملفات الفحص أو التقارير التشغيلية.

## 🔄 ترتيب استخدامه أثناء دورة التطوير
يمثل هذا المجلد **الخطوة الصفرية والأولى دائماً (المرجع الدستوري الأعلى)** في أي دورة تطوير:
1. يرجع إليه المهندس والمطور قبل اتخاذ أي قرار للتحقق من التوافقية الكاملة.
2. عند صياغة أو تصميم نظام أو مرحلة جديدة، يتم مطابقتها كلياً مع القوانين المكتوبة هنا.
3. يمثل المظلة التي يشتق منها نظام فحص الامتثال (`compliance/ArchitectureCompliance.sql`) اختباراته.
