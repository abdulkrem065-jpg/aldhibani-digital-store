## 1. البنية الهيكلية الحالية للنظام (Current Architecture)
يعمل المشروع على بنية هجينة متكاملة (Full-Stack Hybrid Client-Server):
*   **خادم الواجهة الخلفية (Backend Express Server):** 
    *   الملف الأساسي: `server.ts` المترجم إلى `dist/server.cjs` للتشغيل السريع.
    *   يدير بروتوكول قنوات الاتصال والتحقق السكني، بوابات المزامنة واستقبال البيانات عبر `/api/remote-sync/receive` و `/api/sync-products`.
    *   يدير محادثات ومحركات الذكاء الاصطناعي مع نماذج Google Gemini عبر مكتبة `@google/genai`.
*   **الواجهة الأمامية والعميل (Frontend Client SPA):**
    *   التطوير التقني: React 19 + TypeScript + Vite.
    *   تصميم الواجهات والتفاعلات: Tailwind CSS مع تخطيطات حركية دقيقة مدعومة بـ `motion` (مستوردة من `motion/react`).
    *   تفاصيل التصميم البصري: طابع داكن فاخر (Cosmic Obsidian Midnight) بنسب تباين مرتفعة وأيقونات `lucide-react`.

## 2. بنية قاعدة البيانات المستهدفة والمحلية (Current Database Architecture)
تبني المنظومة نمط **"عدم انقطاع الخدمة والعمل دون إنترنت" (Offline-First Local Storage Engine):**
1.  **الطبقة السحابية الاختيارية (Supabase DB Engine):**
    *   يتم قراءة الـ URL والدبابيس الأمنية ديناميكياً من المكونات السحابية.
    *   الجداول المتصلة:
        *   `aldhibani_config`: لحفظ معايير الصرف، والاسم والشرائط التمريرية.
        *   `aldhibani_products`: لبيانات الكتالوج والأسعار.
        *   `aldhibani_categories`: للأقسام الذكية.
        *   `aldhibani_debts`: لسجلات الديون الخارجية والأجل.
        *   `aldhibani_money_boxes`: لحسابات ومطابقة الصناديق.
        *   `aldhibani_youth_workforce`: لقوائم طاقم العمل والعمولات.
2.  **الطبقة المحلية البديلة (Reactive LocalStorage Buffers):**
    *   تتم المزامنة والتخزين المؤقت بشكل تلقائي في متصفح المحاسب تحت وسوم `aldhibani_local_*`. 
    *   في حالة عدم الاتصال بسوبابيس، تنشط واجهة `SupabaseServerlessDB` كقاعدة محلية معتمدة ومستقرة تماماً لمنع توقف البيع والشراء.
