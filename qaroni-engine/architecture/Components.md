# Qaroni AI Engine — Component Specification
## المكونات البرمجية والكتل الهيكلية للمحرك

تتكامل مكونات **Qaroni AI Engine** برمجياً ووظيفياً بطريقة معيارية تمنع التداخل وتحكم سريان القرارات التشغيلية للنظام الحاكم.

---

## 1. المكونات البرمجية والمنطقية الأساسية (The Components)

### أ. محرك تتبع الحالة والحالة المعيارية (State Machine Controller)
* **المسؤولية:** تتبع دورة حياة المهام برمتها من مرحلة الإنشاء وحتى الإتمام التام أو الفشل والتراجع.
* **آلية العمل:** يقوم بحفظ وتحديث مستند الحالة لكل تذكرة برمجية أو هيكلية، ويمنع انتقال أي مهمة لحالة جديدة إلا بعد استيفاء شروط وبوابات الجودة الخاصة بالحالة السابقة.

### ب. النواة الدستورية ومطابقة القواعد (Constitutional Checker & Guard)
* **المسؤولية:** الحارس الداخلي المحكم لكافة التعليمات والاستعلامات والتعديلات المقترحة.
* **آلية العمل:** يعتمد على مسح الكلمات المفتاحية، وفحص هياكل البيانات، ومراجعة نية الاستعلامات والترحيلات في الـ Sandbox لمنع تنفيذ استعلامات تعديل مجهولة المصدر، أو تصفير الجداول، أو محاولة استغلال ثغرات أمنية.

### ج. بوابة الكتابة الآمنة والمطابقة السحابية (Secure Write Gateway)
* **المسؤولية:** القناة والمنفذ الموحد والوحيد المصرح له بكتابة وتحديث البيانات والجداول على الخوادم الحية ومستودعات GitHub.
* **آلية العمل:** يقوم بإعداد الاتصالات المشفرة واستخدام الـ Service Roles المصرحة لتطبيق التحديثات المصدق عليها من المالك بدقة، بعد تجاوز بوابات الفحص بنجاح.

### د. بيئة المحاكاة الافتراضية المعزولة (Isolated Sandbox Simulator)
* **المسؤولية:** عزل عمليات المحاكاة والاختبار وتوليد ترحيلات قواعد البيانات.
* **آلية العمل:** بيئة معالجة مصغرة مستقلة في الذاكرة تستخدم محرك ترحيل افتراضي لمحاكاة التعديل وقياس تأثيره الهيكلي على المخططات وسياسات RLS دون الاقتراب من قاعدة البيانات الحقيقية.

### هـ. المترجم الفوري وبوابات الاتصال (PostgREST Schema Parser)
* **المسؤولية:** قراءة وتحليل وتعديل مخططات OpenAPI والـ Schema Cache المستمدة من Supabase.
* **آلية العمل:** قراءة وفحص ملفات التوثيق والـ Cache واستنباط الجداول والحقول والقيود لبناء نموذج داخلي موحد لهياكل قاعدة البيانات.

### و. مسجل وسجلات التدقيق الفني الموحد (Technical Audit Logger)
* **المسؤولية:** توثيق كل حركة، قرار، موافقة، أو استثناء أمني داخل قاعدة البيانات وفي ملفات تقارير التدقيق.
* **آلية العمل:** يسجل أحداث وجلسات عمل الوكلاء ومحاولات الاستدعاء ومخرجاتها في قاعدة بيانات التدقيق المعزولة لضمان الرجوع إليها لغرض المحاسبة والرقابة والتحقيق الذاتي.

## 2. مصفوفة الترابط وتوجيه المهام (Interaction & Routing Matrix)

| اسم المكون (Component) | المدخلات الرئيسية (Inputs) | المخرجات الرئيسية (Outputs) | قنوات الترابط والاتصال (Channels) |
| :--- | :--- | :--- | :--- |
| **State Machine** | طلبات وتكليفات المهام | الحالة الحالية للمهمة وتحديثها | Event Bus |
| **Constitutional Checker** | الأكواد والترحيلات والخطط المقترحة | قرار المطابقة (Approved / Denied) | State Machine & Sandbox |
| **Write Gateway** | خطط وترحيلات مصدق عليها + توقيع بشري | تنفيذ حقيقي فيزيائي على السيرفر | PostgreSQL TLS & Git HTTPS |
| **Sandbox Simulator** | هياكل مقترحة + كشوفات المخططات الحالية | تقارير المحاكاة ونتائج الفحص والـ Tests | In-Memory Virtual Database |
| **Schema Parser** | بيانات وقوائم الكاش للـ API | الهيكل الهيكلي المستنبط من المخطط | In-Memory Meta-Model |
| **Audit Logger** | كافة حركات وجلسات وعمليات الوكلاء | سجلات وتقارير تدقيق فني شاملة | Write-Only Audit Database |
