# Qaroni AI Engine — Component Specification
## المكونات البرمجية والكتل الهيكلية للمحرك الذاتي المعزول

---

## 1. المكونات البرمجية والمنطقية الأساسية (Core Engine Modules)

يتألف محرك **Qaroni AI Engine** من ثمانية مكونات فنية برمجية معيارية متكاملة تمنع التداخل وتحكم سريان العمليات البرمجية بدقة:

### أ. مراقب ومسؤول الحالات (State Machine Controller)
* **المسؤولية:** الحفاظ على استمرارية العمليات وإدارة الحالات الـ 12 للمهام من مرحلة الإنشاء وحتى الإتمام أو التراجع والإنقاذ.
* **آلية العمل:** يدير ويكتب في السجل المنطقي لقاعدة البيانات `qaroni_engine_state`. يحتوي على محرك استعادة مرن (Recovery Engine) يضمن استئناف العمليات التلقائي من آخر نقطة فحص آمنة (`last_checkpoint`) في حال انقطاع السيرفر أو انقطاع الجلسة.

### ب. النواة الدستورية ومطابقة القوانين (Constitutional Checker & Guard)
* **المسؤولية:** الفحص الأمني السلوكي الصارم لكافة التعليمات والاستعلامات والتعديلات المقترحة لضمان مطابقتها للميثاق الدستوري.
* **آلية العمل:** يقوم بمقارنة طلب العميل ومقترحات الوكلاء ببنود دستور المحرك `EngineConstitution.md` ويقوم بـ "تتبع القرار الدستوري" (Decision Trace) وتسجيل الرمز الحاكم بالشكل: `Constitution (Article X) → ADR-XXX → Specification (Module Y) → Action (Migration Z)`.

### ج. محرك محاكاة البيئة المعزولة وعزل الفروع (Sandbox Simulator & Isolation Engine)
* **المسؤولية:** عزل واختبار التعديلات والترحيلات الهيكلية بعيداً تماماً عن بيئة الإنتاج الحية.
* **آلية العمل:** يدير بيئة العزل المتدرج المادي والمنطقي عبر استدعاء قواعد التحقق المع معرفي (`Knowledge_Validation`) ثم إنشاء فرع مخصص ومستقل لقاعدة بيانات Supabase ليكون مكاناً للفحص المادي الفعلي.

### د. محرك التقييم والتحقق الذاتي (Self-Validation & Testing Engine)
* **المسؤولية:** إجراء الفحوصات الفنية البرمجية والهيكلية على البيئات المعزولة ومطابقة المعايير الفنية بنسبة 100%.
* **آلية العمل:** يقوم بتشغيل فحوصات الصياغة النحوية لـ SQL، تكامل العلاقات والمفاتيح الخارجية (FK)، توفر الفهارس (Indexes)، والتحقق من تمكين سياسات RLS، ومحاكاة عمليات الإدخال والاستعلام الموكتة.

### هـ. محرك التغذية الراجعة والإصلاح التلقائي (Feedback & Repair Loop Engine)
* **المسؤولية:** كشف وتحليل الأخطاء وإصلاحها تلقائياً دون تدخل بشري في مرحلة الاختبار والمحاكاة.
* **آلية العمل:** في حال فشل أي فحص، يستدعي هذا المكون محلل الخطأ لتوليد "تقرير إصلاح" (Fix Report) يحدد الخلل والحل المقترح، ثم يوعز لوكيل الصياغة لإعادة التوليد ومباشرة الاختبار مجدداً (حتى 3 محاولات كحد أقصى).

### و. بوابة الكتابة الآمنة والمطابقة السحابية (Secure Write Gateway)
* **المسؤولية:** المنفذ والممر الوحيد والمشفر المصرح له بكتابة وتحديث البيانات والترحيلات على الخادم الحي لـ Supabase ومستودع GitHub.
* **آلية العمل:** تستخدم المعاملات الفردية المغلقة (Single Transactions) لضمان تطبيق الترحيلات الهيكلية معاً ككتلة واحدة لا تقبل التجزئة، ولا يتم تفعيلها إلا بورود **التفويض البشري الصريح والرمز الرقمي OTP**.

### ز. المترجم الفوري وبوابات الاتصال (PostgREST Schema Parser)
* **المسؤولية:** قراءة وتحليل وتفصيل المخططات والـ Schema Cache المستمدة من Supabase عبر قنوات REST فقط.
* **آلية العمل:** يقوم ببناء نموذج معرفي متكامل لهياكل قاعدة البيانات وتغذية الوكيل المحلل به لمنع الاستعلام العشوائي ومطابقة العلاقات البرمجية بالواقع الفعلي.

### ح. مسجل وسجلات التدقيق والنزاهة الفنية (Technical Audit Logger)
* **المسؤولية:** توثيق كل حركة، قرار، موافقة، فحص، أو تراجع تلقائي برمجياً وحفظها في التقرير الدائم.
* **آلية العمل:** يقوم بجمع تتبع القرارات الدستورية وهوية الوكلاء المعنيين بالعمليات من حافلة الأحداث الداخلية وتصدير التقرير الفني الشامل النهائي لحفظه في مستند التدقيق وحصانته الفنية.

## 2. مصفوفة الترابط وتوجيه المهام والمكونات (Interaction Matrix)

| اسم المكون (Component) | المدخلات الرئيسية (Inputs) | المخرجات الرئيسية (Outputs) | قنوات الترابط والاتصال (Channels) |
| :--- | :--- | :--- | :--- |
| **State Machine** | طلبات وتكليفات المهام | الحالة الحالية للمهمة وكتابة `qaroni_engine_state` | Event Bus & Local Database |
| **Constitutional Checker** | التعديلات والأكواد المقترحة | قرار المطابقة الدستورية ورمز الـ Decision Trace | State Machine & Sandbox |
| **Sandbox Simulator** | هياكل مقترحة + كشوفات المخططات | بيئة عزل مادية وفروع Supabase معزولة | In-Memory & Cloud API |
| **Self-Validation Engine** | اختبارات ومحاكاة العلاقات والـ RLS | نتيجة الاختبار (Succeeded / Failed) | Sandbox & Repair Loop |
| **Feedback & Repair Loop** | تقارير الفشل والأخطاء الفنية | تقرير الإصلاح (Fix Report) وتصحيح الأكواد تلقائياً | Validation Engine & Builder |
| **Write Gateway** | خطط وترحيلات معتمدة + توقيع بشري OTP | تطبيق حقيقي فيزيائي على الإنتاج | PostgreSQL TLS & Git HTTPS |
| **Schema Parser** | بيانات وقوائم الكاش للـ API | الهيكل الهيكلي المستنبط من المخطط | In-Memory Meta-Model |
| **Audit Logger** | كافة حركات وجلسات وعمليات الوكلاء | سجلات وتقارير تدقيق فني شاملة (`Phase1IntegrityAudit.md`) | Write-Only Audit File |
