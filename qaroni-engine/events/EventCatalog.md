# Qaroni AI Engine — Event Catalog Specification
## كتالوج ومواصفات الأحداث الحاكمة للعمليات والمهام بالمحرك

تدار الحركة التشغيلية لجميع وكلاء ومكونات **Qaroni AI Engine** بصورة تفاعلية عبر تتبع ونشر الأحداث (Event-Driven Integration) لضمان اتساق تتبع القرارات البرمجية والهيكلية.

---

## 1. كتالوج الأحداث المعياري (The Event Ingress Catalog)

### 📌 الحدث: `SchemaReadCompleted` (اكتملت معاينة الهيكل)
* **المصدر (Publisher):** الوكيل `Qaroni_Reader`
* **المستلم (Subscriber):** الوكيل `Qaroni_Analyzer` والمكون `State Machine`
* **الوصف التقني:** يُنشر فور نجاح التقصي وقراءة المخططات والـ API لـ Supabase والتأكد من مطابقة كشف الجداول الـ 11 النشطة تشغيلياً.

### 📌 الحدث: `SchemaAnalysisCompleted` (اكتمل التحليل الفني للفجوات)
* **المصدر (Publisher):** الوكيل `Qaroni_Analyzer`
* **المستلم (Subscriber):** الوكيل `Qaroni_MigrationBuilder` والمكون `State Machine`
* **الوصف التقني:** يُنشر فور انتهاء احتساب الفروقات الفنية (Diff) وتدوين الفجوات ورسم شجرة التبعيات بدقة للتعديل المقترح.

### 📌 الحدث: `MigrationCreated` (تم صياغة الترحيل والمسودة)
* **المصدر (Publisher):** الوكيل `Qaroni_MigrationBuilder`
* **المستلم (Subscriber):** الوكيل `Qaroni_Validator` والمكون `State Machine`
* **الوصف التقني:** يُنشر فور الانتهاء من توليد وكتابة المسودة لملفات التحديث (Up Script) والتراجع التلقائي (Down Script) كملف تجريبي داخل الـ Sandbox.

### 📌 الحدث: `ValidationSucceeded` (نجح التحقق والمحاكاة الفنية)
* **المصدر (Publisher):** الوكيل `Qaroni_Validator`
* **المستلم (Subscriber):** المكون `State Machine` والمقيد البشري
* **الوصف التقني:** يُنشر بعد اجتياز ترحيل المحاكاة التراجعي والـ Dry Run بنسبة 100% ومطابقة شروط الـ Linter وقواعد TypeScript وسياسات RLS الأمنية دون توليد أخطاء.

### 📌 الحدث: `DeploymentApproved` (تم تفويض واعتماد التحديث بشرياً)
* **المصدر (Publisher):** المالك/المستخدم البشري (التوقيع الرقمي والموافقة)
* **المستلم (Subscriber):** الوكيل `Qaroni_Executor` والمكون `State Machine`
* **الوصف التقني:** يُنشر فور مصادقة المالك الرقمية وتأكيد رغبته الصريحة في بث وتنزيل التعديلات المقترحة على خادم الإنتاج السحابي.

### 📌 الحدث: `DeploymentCompleted` (اكتمل البث والتطبيق الفعلي للإنتاج)
* **المصدر (Publisher):** الوكيل `Qaroni_Executor`
* **المستلم (Subscriber):** الوكيل `Qaroni_Auditor` والمكون `State Machine`
* **الوصف التقني:** يُنشر بعد إتمام عمليات ترحيل الجداول في Supabase ودمج الأكواد بمستودع GitHub للإنتاج بنجاح تمهيداً لتشغيل الفحص البعدي.

### 📌 الحدث: `RollbackStarted` (تم تفعيل بروتوكول الإنقاذ والتراجع الطارئ)
* **المصدر (Publisher):** الوكيل `Qaroni_Auditor` أو المالك يدوياً
* **المستلم (Subscriber):** الوكيل `Qaroni_Executor` والمكون `State Machine`
* **الوصف التقني:** يُنشر فور استشعار عطل تقني بعد النشر أو حدوث استثناءات أمنية، لبدء عملية إعادة الجداول والأكواد للحالة السابقة تلقائياً لحفظ استقرار المتجر.

### 📌 الحدث: `RollbackCompleted` (اكتمل التراجع بنجاح وأعيد الاستقرار)
* **المصدر (Publisher):** الوكيل `Qaroni_Executor`
* **المستلم (Subscriber):** الوكيل `Qaroni_Auditor` ومجلس الحوكمة
* **الوصف التقني:** يُنشر فور نجاح تطبيق ملفات التراجع وسحب التعديل المعطوب وإعادة المعرض لحالة العمل المستقرة السابقة.

### 📌 الحدث: `AuditCompleted` (اكتمل التدقيق الرقابي وإغلاق المهمة)
* **المصدر (Publisher):** الوكيل `Qaroni_Auditor`
* **المستلم (Subscriber):** المكون `State Machine` ومجلد التقارير
* **الوصف التقني:** يُنشر فور انتهاء التدقيق الفوري الشامل، وكتابة التقرير الفني النهائي لسلامة النظام، وحفظ السجلات في مجلد التقارير لإنهاء المهمة رسمياً.

## 2. بنية البيانات النموذجية للأحداث (Standard Event Schema Spec)
تحفظ الأحداث برمجياً داخل حافلة المهام ككائنات مهيكلة غير قابلة للتعديل وتحتوي على التفاصيل الحاكمة التالية:
* `event_id`: معرف فريد للحدث (UUID).
* `task_id`: معرف المهمة والترحيل المرتبط (UUID).
* `event_type`: اسم نوع الحدث من القائمة أعلاه.
* `timestamp`: وقت الصدور الفعلي وفق توقيت السيرفر.
* `publisher`: الوكيل المنشئ للحدث.
* `payload`: كائن تفصيلي يحمل البيانات، التوقيع الرقمي للموافقة، أو نصوص الأخطاء والاستثناءات المرتبطة بالحدث لتسهيل التدقيق والتحليل.
