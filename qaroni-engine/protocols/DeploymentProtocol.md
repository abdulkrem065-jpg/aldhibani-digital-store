# Qaroni AI Engine — Deployment Protocol Specification
## بروتوكول البث الحي وتطبيق الأكواد والترحيلات الهيكلية الحية

---

## 1. الغرض والأهداف (Purpose & Scope)
يهدف هذا البروتوكول لحوكمة عملية النشر والتحديث الفعلي والتنزيل لملفات الأكواد وتطبيق ترحيلات الجداول (Database Migrations) على خوادم الإنتاج لـ Supabase ومستودعات GitHub الحية بطريقة تضمن استقرار النظام واستمرارية عمل المعرض بدون انقطاع للزبائن.

---

## 2. القيود وشروط تفعيل النشر (Deployment Pre-conditions)
* **إلزامية التوقيع والتحقق الرقمي (OTP / User Consent):** يمنع البروتوكول تطبيق أي ترحيل هيكلي في السيرفر الحي دون استقبال ومعالجة كود الموافقة الرقمي المصدق عليه من المالك.
* **البث بنمط المعاملة الفردية المعزولة (Transactional Isolation):** تُطبق التحديثات الهيكلية على قاعدة بيانات Supabase كحزمة واحدة منعزلة (Single Transaction Blocks) وفي فترات انخفاض حركة الشراء، لضمان استرجاع النظام بالكامل للحالة السابقة تلقائياً في حال فشل أي جزء من العملية.
* **النشر التراكمي وتحديث الـ Cache في سوبابيز:** عند إضافة جدول أو حقل جديد، يجب تصفير وتحديث ذاكرة الـ Schema Cache لـ PostgREST تلقائياً لتمكين بوابات الـ API من التعرف على المكونات الجديدة فوراً وتجنب ظهور أخطاء `404 Not Found`.

---

## 3. خطوات بروتوكول البث والتطبيق (Execution Steps)
1. **الخطوة الأولى: التحقق من التوقيع البشري والموافقات البرمجية:**
   * استقبال ومطابقة التوقيع والرمز الأمني للمالك والموافقة البرمجية وتدوينها بسجلات النظام.
2. **الخطوة الثانية: عزل بيئة البث المباشر (Production Lockdown/Isolation):**
   * تجهيز قنوات الاتصال المشفرة بـ TLS وبوابة الكتابة `WriteGateway` وضبط النظام لاستقبال التحديث بهدوء وأمان.
3. **الخطوة الثالثة: تطبيق التغييرات الهيكلية (Apply Schema Changes):**
   * تنفيذ عمليات ترحيل الجداول الهيكلية على Supabase، ومطابقة العلاقات، وتدشين سياسات RLS الأمنية المصاحبة فوراً.
4. **الخطوة الرابعة: دمج الأكواد ورفع التحديثات (Git Merge & Code Push):**
   * دمج الأكواد المختبرة في مستودع GitHub وبناء وتحديث حزم الخادم والواجهات الأمامية بنجاح.
5. **الخطوة الخامسة: نشر الحدث ونقل الحالة للفحص البعدي:**
   * تدوين نجاح النشر ونشر الحدث المعياري `DeploymentCompleted` ونقل المهمة لحالة `Verifying` لتشغيل الفحوصات التشغيلية البعدية.
